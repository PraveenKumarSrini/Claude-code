# URL Shortener — TDD Exercise

Practice Test-Driven Development using the Red-Green-Refactor cycle.

## Stack
- TypeScript + Vitest for testing
- nanoid for short code generation

## TDD Rules
1. Write tests FIRST — before any implementation
2. Run tests to see them FAIL (RED)
3. Implement the minimum code to PASS (GREEN)
4. Refactor while keeping tests passing (REFACTOR)

## Project Structure
- `src/shortener.ts` — The URL shortener module (to be implemented)
- `src/shortener.test.ts` — Tests (to be written FIRST)

## Important
Do NOT write implementation and tests at the same time.
Follow the Red-Green-Refactor cycle strictly.
